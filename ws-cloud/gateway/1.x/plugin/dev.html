# 插件开发

要开发一个插件非常简单：

  1. 在 gateway/app/plugins 目录下创建一个lua文件，如：demo_plugin.lua

  2. 在各钩子函数中编写对应方法，如果在对应阶段不需要处理可以不定义该阶段对应的钩子函数。各阶段钩子函数参考文档：https://moonbingbing.gitbooks.io/openresty-best-practices/content/ngx_lua/phase.html

    -- gateway/app/plugins/demo_plugin.lua

 local _M = {
     name = "demo",
     desc = "测试插件",
     version = 1.0
 }

 -- 对应 init 阶段
 function _M.do_in_init(route)
     -- body
 end

 -- 对应 init_worker 阶段
 function _M.do_in_init_worker(route)
     -- body
 end

 -- 对应 rewrite 阶段
 function _M.do_in_rewrite(route)
     -- body
 end

 -- 对应 access 阶段
 function _M.do_in_access(route)
     -- body
 end

 -- 对应 content 阶段
 function _M.do_in_content(route)
     -- body
 end

 -- 对应 balancer 阶段
 function _M.do_in_balancer(route)
     -- body
 end

 -- 对应 body_filter 阶段
 function _M.do_in_body_filter(route)
     -- body
 end

 -- 对应 log 阶段
 function _M.do_in_log(route)
     -- body
 end

 return _M

    参数 route 是一个 table

    route = {
    prefix
    status
    service_name
    protocol
    plugins
}

    需要注意的是，init 和 init_worker 阶段不能发起网络请求，需要在 ngx.timer 中执行，这个是 openresty 本身的限制

# 插件注册

修改 conf/app.json，在 plugins 字段中加上插件对应的文件名（不用加后缀）


{
    "plugins": [
        "default_plugin",
        "discovery_plugin",
        "tracing_plugin",
        "rewrite_plugin",
        "demo_plugin"  // 注册 demo_plugin 插件
    ]
}

# 插件配置

目前插件配置还不支持在后台面板中设置（开发中），需要手动将配置写到 etcd 中，参考 t/app/init/etcd_dada.t 的 === TEST 1: init route，在 pipelined_requests eval 中增加加 request 就可以了（注意同时需要增加 response_body eval）

{
    "prefix": "/openapi/demo1",
    "status": 1,
    "service_name": "demo1",
    "protocol": "http",
    "plugins": [
        "discovery",
        "tracing",
        "rewrite"
    ],
    "props": {
        "rewrite_url_regex" : "^/openapi/(.*)/",
        "rewrite_replace" : "/openapi/"
    }
}

参数解析：

  • prefix: 匹配路由前缀
  • status: 1 - 有效；0 - 无效
  • service_name: 服务名，上游服务节点会通过服务名来查找
  • protocol: 协议类型（目前支持 http 和 grpc）
  • plugins: 路由启用的插件
  • props: 路由参数配置，插件可以读取参数配置执行对应逻辑（例如: rewrite 插件）
